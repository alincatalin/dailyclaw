### Setup â€” Authentication

1. Download your Google Cloud OAuth credentials (Desktop app type):
   ```bash
   gog auth credentials ~/Downloads/client_secret_....json
   ```
2. Add an account (opens browser for OAuth consent):
   ```bash
   gog auth add you@gmail.com
   ```
3. Set your default account:
   ```bash
   export GOG_ACCOUNT=you@gmail.com
   ```

Credentials are stored in your OS keyring (Keychain on macOS, Secret Service on Linux, CredMan on Windows). Use `gog auth manage` to switch between multiple accounts.

### Setup â€” Gmail Pub/Sub with OpenClaw

GOG's `gmail watch serve` daemon connects Gmail push notifications to [OpenClaw webhooks](https://docs.openclaw.ai/automation/gmail-pubsub), enabling real-time email processing and delivery to chat surfaces.

**How it works:** Gmail watch detects new messages â†’ publishes to Pub/Sub topic â†’ `gog gmail watch serve` daemon receives push â†’ forwards to OpenClaw webhook â†’ processes email and delivers summary.

**Quick setup (recommended):**
```bash
openclaw webhooks gmail setup --account openclaw@gmail.com
```

This installs dependencies, configures Tailscale Funnel for HTTPS tunneling, and enables the Gmail hook preset automatically.

**Manual setup:**
1. Configure your GCP project and enable Gmail + Pub/Sub APIs:
   ```bash
   gcloud services enable gmail.googleapis.com pubsub.googleapis.com
   ```
2. Create the Pub/Sub topic:
   ```bash
   gcloud pubsub topics create gog-gmail-watch
   ```
3. Grant Gmail API publish permissions to the topic
4. Run the daemon:
   ```bash
   gog gmail watch serve --account you@gmail.com \
     --bind 127.0.0.1 --port 8788 --path /gmail-pubsub \
     --hook-url <openclaw-endpoint>
   ```

Or use the automated runner: `openclaw webhooks gmail run`

### Example usage

```bash
# Search Gmail
gog gmail search "from:sarah subject:roadmap" --max 5

# Send an email
gog gmail send --to team@company.com --subject "Standup notes" --body "..."

# List today's calendar events
gog calendar list --from today --to tomorrow

# Upload a file to Drive
gog drive upload ./report.pdf --folder "Shared Reports"

# Read a Google Sheet as JSON
gog sheets get <sheet-id> --format json

# Check Gmail watch status
gog gmail watch status --account you@gmail.com
```

### Tips for better results

Use `--format json` when piping output to agents â€” it gives structured data instead of table formatting. For multi-account setups, set `GOG_ACCOUNT` per session or use `gog auth manage` to switch. The Gmail Pub/Sub watch auto-renews, but check status with `gog gmail watch status` if notifications stop. Gmail hook content in OpenClaw is wrapped with safety boundaries by default to prevent injection attacks via email content.
